## MarkdownPost

**Props:**

- content: string -> Markdown content

Renders markdown using react-markdown & react-syntax-highlighter

Usage:

```jsx
<MarkdownPost content={content} />
```

## Modal

- Modal -> Base for a modal
- CloseIcon -> Close icon for the modal

### Modal -> BaseComponent

**Props:**

- isOpen: boolean -> Whether the modal is open or not
- onClose: () => void -> Function to call when the modal is closed
- closeIcon: boolean -> Whether to show the close icon or not
- variant: "base" | "fullScreen" -> Size of the modal(default: base)
- children: React.ReactNode -> Children to render inside the modal

Usage:

```jsx
<Modal isOpen={isOpen} onClose={onClose} closeIcon={true} variant={"base"}>
  {children}
</Modal>
```

### CloseIcon

**Props:**

- onClose: () => void -> Function to call when the close icon is clicked

Usage:

```jsx
<CloseIcon onClose={onClose} />
```

## Navigation

### Link

> Important: When documenting a component that uses the Link atom, make sure to use the `docs` prop on the Link atom. This will make sure that the link is styled correctly and prevent the use of the BrowserContext outside the BrowserProvider.

**Props:**

- content: string -> Content to render inside the link
- to: string -> Path to navigate to
- color?: string -> Color of the link
- variant?: keyof typeof variants -> Variant of the link
- as?: keyof typeof asEnum -> HTML tag to render the link as
- docs?: boolean -> Whether the link is a docs link or not
